전제 / 도출 기준 ✓ v0.3 메인·탭 구조 확정

  • 프레임 — Next.js 기반 모바일 우선 반응형 웹앱.
  • 메인 화면 = 4개 친숙어 CTA — "나는 누구일까? / 남이 보는 내 모습 / 우리 잘 맞을까? / 이 서비스는 무엇인가요?". 연령대와 무관하게 기능을 단번에 이해하는 것을 목표 (확정).
  • 하단 탭바 4개 — 홈 / 검사 결과 / 동료 & 케미 / 내 정보. 회원·비회원 모두에게 노출되며, 비회원이 잠긴 항목을 탭하면 "로그인이 필요해요" 토스트 안내 (확정 · v0.2의 "부가 기능" → "내 정보"로 명칭 변경).
  • 메인 CTA ↔ 하단 탭 — 같은 목적지로 이동. CTA는 친숙한 진입, 탭은 빠른 진입(보조 동선). 두 진입점이 같은 화면을 가리키도록 유지.
  • 검사 응시 = 풀스크린 시트 — `/test` 라우트 없음. 메인 위에 시트로 오버레이. Step 1(유형 선택)은 자유롭게 닫기 가능, Step 2(응시 중) 이후 닫기 시 확인 다이얼로그 (확정).
  • 공유(타인 평정) 모달 — 검사 모달과 동일 컴포넌트를 재사용. Step만 분기 — 유형 선택 후 "응시" 대신 초대 링크 생성 + 카카오톡 공유. 회원 전용 (확정).
  • 마이페이지 진입 — 헤더 프로필 칩 + 하단 "내 정보" 탭 첫 항목 — 두 경로 모두 진입 가능 (확정).
  • 모달 6종 — 검사 응시(풀스크린 시트) / 타인 평정 공유(동일 컴포넌트) / 동료 등록 / 케미 발행 확인 / 서비스 소개 시트 / 알림 드롭다운. 코인 잔량 안내는 케미 발행 확인 모달에 통합.
  • 케미 비동기 발행 — 발행 즉시 "케미 보고서 발행 중입니다…" 토스트 1회 노출 후 자동 사라짐 → SSE 완료 이벤트 도착 시 토스트 + 알림 종 뱃지 동시 표시 (확정).
  • 전역 요소 — 상단 헤더(로고 · 알림 종 · 프로필 칩 또는 카카오 로그인 버튼), 하단 탭바, SSE 토스트 영역, 푸터(면책 조항: "참고용 성향 분석, 심리 진단 아님").

주요 동선

User Journeys

유즈케이스 문서의 UC-01 / UC-02 / UC-03 + 신규 보강 동선 1개. 각 단계의 볼드된 키워드는 위의 화면 카탈로그와 1:1 대응.

① 비회원 → 검사 → 회원 전환

UC-01
  1. 메인에서 "나는 누구일까? (심리검사)" CTA 클릭 → 풀스크린 시트 오픈 @ 메인 → /
  2. Step 1 — 검사 유형 선택 (DISC / MBTI·Big5는 "준비 중"). 시트 자유 닫기 가능 @ 검사 시트 · Step 1
  3. Step 2 — 24문항 강제선택 응시 (Most/Least). 닫기 시 확인 다이얼로그 @ 검사 시트 · Step 2
  4. Step 3 — 보고서 표시. 비회원은 sessionStorage에 원점수 임시 보관 + "카카오로 결과 저장" CTA 강조 @ 검사 시트 · Step 3
  5. 카카오 OAuth 콜백 → sessionStorage 원점수를 POST /results → 결과 상세로 이동 @ /auth/kakao → 결과 상세 → /results/[id]

② "남이 보는 내 모습" → 카카오 공유

UC-02
  1. 메인 CTA ② 클릭 → 검사 모달과 동일 컴포넌트(시트) 오픈, Step만 분기 @ 메인 → /
  2. Step 1 — 검사 유형 선택 (응시 모달과 동일 UI) @ 공유 시트 · Step 1
  3. Step 2 — 라벨 입력 ("여자친구" 등). 일회용 토큰 발급 @ 공유 시트 · Step 2 → POST /assessments
  4. Step 3 — 링크 발급 화면. 카카오톡 공유 버튼 활성화 / URL 복사 @ 공유 시트 · Step 3 · Kakao Share SDK
  5. 대상자(비회원 가능) 링크 접속 → 응시 → rater=OTHER로 A 계정에 귀속 → "친구가 본 내 모습" 탭에서 조회 @ 타인 평정 응시 → /assessments/[token]

③ 케미 발행 → 토스트 → SSE 완료

UC-03
  1. 동료 프로필에서 "케미 발행" → 코인 차감 확인 모달 → 확인 @ 동료 프로필 → /colleagues/[id]
  2. POST /chemistry-reports → 202 즉시 반환 → 즉시 "케미 보고서 발행 중입니다…" 토스트 1회 노출 후 자동 사라짐 @ SSE 토스트 영역 · ~3s auto-dismiss
  3. 사용자는 자유롭게 다른 화면 이동 가능 (메인·결과·동료 등). 발행 중 상태는 /chemistry 이력에 카드로 표시 @ 케미 이력 → /chemistry · "발행 중" 카드
  4. SSE 완료 이벤트 수신 → 화면 위치 무관하게 토스트 ("케미 보고서가 도착했어요") + 헤더 알림 종 뱃지 동시 표시 @ SSE /notifications/stream · type=chemistry.completed
  5. 토스트의 "보기" 또는 알림 종 드롭다운 클릭 → 케미 보고서 상세 → 알림 자동 삭제 @ 케미 보고서 상세 → /chemistry/[id]

④ 비회원 잠긴 CTA → 로그인 유도

신규
  1. 비회원 상태로 메인 진입. 헤더 우측은 "카카오로 시작" 버튼 @ 메인 → / (guest)
  2. CTA ② 또는 ③, 또는 하단 탭(검사결과 / 동료 & 케미 / 내 정보)을 탭 — 항목은 opacity 0.5 + 자물쇠 아이콘으로 표시 @ locked button / tab
  3. 클릭 시 하단에서 토스트 슬라이드 업: "🔒 로그인이 필요해요" + "카카오로 시작" 액션 @ 토스트 · ~4s + tap to dismiss
  4. 토스트 액션 또는 헤더 버튼 탭 → 카카오 OAuth → 콜백 후 직전 의도 화면으로 라우팅 (returnTo) @ /auth/kakao · returnTo=…
  5. 로그인 완료 후 자물쇠 해제. 동일 화면에서 정상 동선 진입 @ 직전 의도 화면

다음 단계 — 와이어프레임 우선순위

확정된 IA(v0.3)를 기준으로 모바일 우선으로 아래 순서에 따라 3~5안씩 발산하기를 제안합니다.

  1. P1
    메인 (4-CTA + 하단 탭바)친숙어 4-CTA의 시각적 리듬, 잠김 표현, 탭바의 단령 표현. 메인에서 모든 동선이 시작.
  2. P1
    검사 풀스크린 시트 (3 Step)서비스의 본체. Step 1 자유 닫기 / Step 2 확인 다이얼로그 / Step 3의 분기 처리.
  3. P1
    공유 시트 (검사 모달 재활용)결과적으로 같은 컴포넌트가 다른 Step 세트를 렌더하는 구조. 재사용 패턴 확정.
  4. P2
    마이페이지 (리스트 계층형)첨부 모형 그대로 — 4개 항목만 노출. 최소한의 구조.
  5. P2
    동료 목록 + 발행 토스트 + SSE 완료 알림단계별 토스트 정책 검증. 발행 중 / 완료 시각적 차별화.
  6. P3
    결과 이력 / 인사이트친숙어 탭 명칭 적용 ("내 검사 결과" / "친구가 본 내 모습").

v0.3 확정 사항

  • 메인 = 4-CTA(친숙어) + 하단 탭바 4개 (홈 / 검사 결과 / 동료 & 케미 / 내 정보)
  • 하단 탭은 모두에게 노출, 비회원은 잠김으로 표시 → 탭 시 토스트
  • 마이페이지 진입 = 헤더 칩 + "내 정보" 탭 첫 항목 공존
  • 공유 모달은 검사 모달과 동일 컴포넌트, Step만 분기
  • 로그아웃 즉시 / 탈퇴 사유 + 2단 확인
  • 인사이트는 "내 정보" 하위 항목으로 이동
  • 결과 이력 탭은 "내 검사 결과" / "친구가 본 내 모습" 친숙어
  • 케미 발행 = 즉시 토스트 1회 → SSE 완료 토스트 + 알림 뱃지
