## 빈 상태 (Empty State) — 데이터가 없을 때

| 화면                               | 조건                      | 표시                                                 | CTA                                       |
| ---------------------------------- | ------------------------- | ---------------------------------------------------- | ----------------------------------------- |
| `results-list` (내 검사 결과)      | 검사 0회                  | 일러스트 placeholder + "아직 검사 기록이 없어요"     | "지금 검사 시작" → `test-sheet-step1`     |
| `results-list` (친구가 본 내 모습) | 타인 평정 0회             | "아직 친구에게 평정 요청한 적이 없어요"              | "친구에게 요청하기" → `share-sheet-step1` |
| `peers-list`                       | 동료 0명                  | "아직 등록된 동료가 없어요" + 내 동료 코드 카드 강조 | "친구 코드 입력" → `peer-register-modal`  |
| `chemistry-list`                   | 케미 보고서 0건           | "아직 발행한 보고서가 없어요"                        | "동료 보러 가기" → `peers-list`           |
| `me-notifications`                 | 알림 0건                  | "아직 알림이 없어요"                                 | —                                         |
| `me-coins` 사용 이력               | 이력 0건                  | 카운트다운 + 슬롯 + "이용 이력이 없어요"             | —                                         |
| `peer-detail` 이전 케미            | 0건                       | "이 동료와의 첫 케미예요"                            | "케미 발행"                               |
| `me-insights-trend`                | 검사 1회만                | "한 번 더 검사하면 변화 추이를 볼 수 있어요"         | "검사 시작"                               |
| `me-insights-compare`              | 비교 불가 (프로필 미완성) | "프로필을 완성하면 비교 통계를 볼 수 있어요"         | "프로필 완성" → `me-profile`              |

**디자인 원칙**

- 빈 상태에는 **반드시 다음 행동을 제안**하는 CTA 포함
- 일러스트는 placeholder OK (실제 일러스트는 추후 추가)
- 톤은 친절하게, 사용자 잘못이 아님을 명시

## 에러 상태 (Error State) — API/네트워크 실패

| 패턴                                   | 사용처                                         | UI                                                                              |
| -------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------------------------- |
| **재시도 가능한 에러** (5xx, 네트워크) | 모든 GET                                       | 페이지 영역에 "잠시 후 다시 시도해 주세요" + 재시도 버튼                        |
| **권한 에러** (401)                    | 회원 전용 화면을 비회원이 접근                 | 로그인 페이지로 자동 리다이렉트 (with `returnTo`)                               |
| **권한 에러** (403)                    | 본인 데이터가 아닌 결과/보고서                 | "접근 권한이 없어요" + 홈으로                                                   |
| **리소스 없음** (404)                  | `/results/[id]`, `/colleagues/[id]` 등         | "결과를 찾을 수 없어요" + 이전 화면으로                                         |
| **검증 실패** (400)                    | POST/PATCH                                     | 필드 인라인 에러 + 토스트                                                       |
| **충돌** (409)                         | `POST /colleagues` 본인 코드, 이미 등록된 동료 | 모달 내 인라인 메시지                                                           |
| **차감 실패**                          | 코인 0개에 케미 발행 시도                      | 발행 버튼 비활성 (모달 진입 자체 차단)                                          |
| **비동기 실패**                        | 케미 생성 실패                                 | 토스트 "발행에 실패했어요. 코인이 환불되었어요." + chemistry-detail에 에러 상태 |

## 화면별 특수 상태

### `assessment-intro` (외부 평정 진입)

| 조건               | 표시                                                     |
| ------------------ | -------------------------------------------------------- |
| 정상 토큰          | 평정자 환영 + 시작 CTA                                   |
| 만료된 토큰        | "초대장이 만료됐어요. 보낸 사람에게 다시 요청해 주세요." |
| 이미 사용된 토큰   | "이미 응시가 완료된 링크예요."                           |
| 존재하지 않는 토큰 | "잘못된 링크예요."                                       |

### `invite-accept` (동료 초대 수락)

| 조건          | 표시                                             |
| ------------- | ------------------------------------------------ |
| 정상 (비회원) | 초대자 정보 + 카카오 로그인                      |
| 정상 (회원)   | "민지님을 동료로 등록하기" CTA                   |
| 만료 (24시간) | "초대장이 만료됐어요"                            |
| 본인 코드     | "본인의 초대장이에요"                            |
| 이미 등록됨   | "이미 동료로 등록되어 있어요" + 동료 상세로 이동 |

### `chemistry-detail`

| 조건                | 표시                                                                             |
| ------------------- | -------------------------------------------------------------------------------- |
| `generating`        | hero(두 아바타) + 스피너·스켈레톤. SSE 도착 시 자동 전환                         |
| `ready`             | 정상 본문                                                                        |
| `error` (생성 실패) | "발행에 실패했어요. 코인이 환불되었어요." + 재시도                               |
| 상대 탈퇴           | hero에 "이 동료는 탈퇴했어요" 라벨, 본문은 정상 노출 (이미 발행된 보고서는 유지) |

### `peer-detail`

| 조건                 | 표시                                                          |
| -------------------- | ------------------------------------------------------------- |
| 정상                 | 프로필 hero + 발행 CTA                                        |
| 코인 0               | 발행 CTA 비활성 + "다음 충전까지 23:14"                       |
| 상대가 검사 미완     | 발행 CTA 비활성 + "민지님이 아직 검사를 완료하지 않았어요"    |
| 상대 탈퇴 (orphaned) | "이 동료는 탈퇴했어요" + 이전 케미 보고서만 노출, 발행 비활성 |
| 본인 검사 미완       | "내 검사 결과가 필요해요" + 검사 시작 CTA                     |

### `share-sheet-step3`

| 조건       | 표시                       |
| ---------- | -------------------------- |
| 생성 중    | 스피너 + "링크 만드는 중…" |
| 성공       | QR + URL + 카카오 공유     |
| 실패 (5xx) | 재시도 버튼                |

### `test-sheet-step3-guest` → 카카오 저장

| 조건               | 표시                                    |
| ------------------ | --------------------------------------- |
| 저장 성공          | 자동으로 `result-detail`로 이동         |
| 카카오 로그인 취소 | Step 3 비회원 화면으로 복귀, 결과 유지  |
| 저장 실패          | "저장에 실패했어요. 다시 시도해 주세요" |

## 로딩 상태 (Loading State)

| 패턴                     | 사용처                         | UI                                                          |
| ------------------------ | ------------------------------ | ----------------------------------------------------------- |
| **페이지 진입 스켈레톤** | 모든 GET이 1회 발생하는 페이지 | 카드 골격을 회색 블록으로                                   |
| **인라인 로더**          | 폼 제출, 모달 액션             | 버튼 내 스피너 + 텍스트 변경 ("저장 중…")                   |
| **풀스크린 인터스티셜**  | OAuth 콜백                     | `kakao-callback` 전용 화면                                  |
| **비동기 발행 대기**     | 케미 보고서                    | 같은 자리에서 점선 카드 + 스켈레톤 본문 (`generating` 상태) |

## 권한 부족 시 UX (회원 전용 진입)

비회원이 회원 전용 CTA를 탭한 경우:

1. **즉시 토스트** (검정 배경, 카카오 액션 포함): "🔒 로그인이 필요해요 [카카오로 시작]"
2. 카카오 액션 탭 → 로그인 흐름 → 콜백 성공 시 원래 가려던 곳으로 자동 진입
3. 토스트 무시 + 다른 곳 이동 → 정상 동작

비회원이 회원 전용 URL을 **직접 입력**한 경우:

1. 미들웨어에서 `/auth/kakao`로 리다이렉트 (returnTo 파라미터 포함)
2. 로그인 성공 시 원래 URL로
