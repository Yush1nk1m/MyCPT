## 1. 서비스 개요

### 1.1 서비스명

**MyCPT** (나의 역량, My ComPeTency)

### 1.2 서비스 개요

DISC 이론 기반의 24문항 강제선택 검사를 통해 나의 역량을 분석하고, 동료와의 협업 케미에 대한 보고서를 제공받을 수 있는 무료 역량 분석 서비스

### 1.3 서비스 배경

직무 역량에 대한 관심이 높아지고 있는 반면, 기존 DISC 검사 도구들은 유료로 제공되고 있어 일반인의 접근성이 낮다. MyCPT는 LLM 기술을 활용해 DISC 검사 척도를 무료로 제공하고, 개인별 맞춤형 피드백 및 동료와의 케미 분석 기능을 제공함으로써 사용자가 자신의 역량을 파악하고 협업 과정에 활용할 수 있도록 지원한다.

### 1.4 서비스 목표

- 누구나 부담 없이 자신의 역량을 파악할 수 있도록 무료로 서비스 제공
- DISC 이론에 기반한 신뢰성 있는 분석 결과 제공
- 협업 동료 간 상호 이해 증진

## 2. 핵심 가치 및 차별점

### 2.1 핵심 가치

| 가치       | 설명                                           |
| ---------- | ---------------------------------------------- |
| **접근성** | 무료 제공, OAuth 간편 가입                     |
| **신뢰성** | DISC 이론 기반 검증된 측정 방식(강제선택) 적용 |
| **개인화** | LLM 기반 성향 맞춤 강점/약점 피드백            |
| **관계성** | 동료 케미 분석으로 협업 인사이트 제공          |

### 2.2 기술적 차별점

LLM API의 사용량에 상한을 두기 위해 **버킷 기반 캐싱 전략**을 적용하여 비용을 최소화한다.

- DISC 각 척도를 6단계로 정규화 → `6⁴ = 1,296`가지 버킷
- 동일 버킷의 결과는 DB에서 즉시 반환 (캐시 히트)
- 신규 버킷 조합에 한해 LLM API를 1회 호출하고 결과를 저장 (캐시 미스)
- 이론상 최대 **1,296회 API 호출**로 전체 사용자 커버 가능
- 1~2개월 주기 캐시 리프레시를 통해 분석 결과를 유동적으로 변화

## 3. 사용자 정의

### 3.1 비회원

- 카카오 로그인 없이 검사 응시 가능
- 검사 응시 횟수 제한 없음 (LLM 캐싱으로 비용 방어)
- 검사 완료 후 나이(출생연도), 성별 입력 필요
- 통계 반영 및 결과 저장은 로그인 후에만 가능
- '로그인하고 결과 저장' 버튼을 통해 카카오 로그인 시 결과 즉시 저장

### 3.2 회원 (카카오 로그인)

- 검사 응시 횟수 제한 없음
- 검사 결과 이력 저장 및 조회
- 나이대/성별 기준 타인 통계 비교
- 동료 코드 생성 및 케미 보고서 발행 이용 가능
- 코인 시스템을 통한 동료 케미 보고서 발행 횟수 관리

## 5. 비기능 요구사항

### 5.1 성능

- 캐시 히트 시 결과 응답 시간 **1초 이내**
- LLM 호출 시 결과 응답 시간 **10초 이내** (로딩 UX 제공)

### 5.2 보안

- 카카오 수집 정보는 서비스 목적 외 사용 금지
- 개인정보처리방침 동의 후 서비스 이용 가능
- DISC 분석 결과는 민감 정보로 취급하여 암호화 저장

### 5.3 확장성

- 버킷 캐시 구조로 사용자 증가 시에도 LLM 비용 선형 증가 없음
- 통계 데이터 증가에 대비한 집계 테이블 별도 관리

### 5.4 운영

- 캐시 초기화: 배치 스케줄러로 1~2개월 주기 자동 실행
- 서비스 면책 조항 명시: "본 결과는 참고용 성향 분석이며 심리 진단이 아닙니다"

## 7. 검사 설계

### 7.1 측정 이론

DISC(Dominance · Influence · Steadiness · Conscientiousness) 이론은 William Moulton Marston이 제안한 행동 성향 모델로, 4가지 차원에서 개인의 행동 패턴을 측정한다. 이론 자체는 퍼블릭 도메인이며, MyCPT는 독자적인 브랜드명과 문항으로 구성된 서비스임을 명시한다.

### 7.2 측정 방식

**강제선택(Forced Choice)** 방식을 채택한다.

- 1문항당 4개의 보기 제시 (각각 D/I/S/C 성향 대응)
- 사용자는 "나와 가장 가까운 것(Most)"과 "가장 먼 것(Least)"을 각각 1개씩 선택
- 사회적 바람직성 편향(social desirability bias)을 줄이고 상대적 성향 강도를 측정

### 7.3 문항 구성

| 척도                  | 의미                     | 문항 수       |
| --------------------- | ------------------------ | ------------- |
| D (Dominance)         | 주도형: 결단, 도전, 통제 | 24문항에 분산 |
| I (Influence)         | 사교형: 설득, 열정, 표현 | 24문항에 분산 |
| S (Steadiness)        | 안정형: 인내, 협력, 루틴 | 24문항에 분산 |
| C (Conscientiousness) | 신중형: 정확, 분석, 완벽 | 24문항에 분산 |

총 24문항, 각 문항에 D/I/S/C 보기 1개씩 포함

### 7.4 채점 방식

```
Most 선택  → 해당 척도 +2점
Least 선택 → 해당 척도 -1점
미선택     → 0점

원점수 범위: -6 ~ +12 (24문항 기준)
```

### 7.5 점수 정규화 및 버킷 분류

원점수(-6 ~ +12, 범위 18)를 6단계로 정규화한다.

| 버킷 | 원점수 범위 | 의미      |
| ---- | ----------- | --------- |
| 1    | -6 ~ -4     | 매우 낮음 |
| 2    | -3 ~ -1     | 낮음      |
| 3    | 0 ~ 2       | 보통 이하 |
| 4    | 3 ~ 5       | 보통 이상 |
| 5    | 6 ~ 8       | 높음      |
| 6    | 9 ~ 12      | 매우 높음 |

D/I/S/C 각 척도가 6단계 → 조합 수: **6⁴ = 1,296가지**

## 8. LLM 비용 최적화 전략

### 8.1 개인 분석 결과 — Lazy Caching

```
사용자 응답 완료
       ↓
D/I/S/C 원점수 산출 → 버킷 정규화 (각 1~6)
       ↓
DB SELECT WHERE d=? AND i=? AND s=? AND c=?
       ↓
캐시 HIT ──→ 즉시 반환 (LLM 호출 없음)
캐시 MISS
       ↓
Claude API 호출 (프롬프트: 버킷 조합 기반 성향 분석 요청)
       ↓
DB INSERT (d, i, s, c, result, created_at)
       ↓
사용자에게 반환
```

- 최대 **1,296회** API 호출로 모든 경우의 수 커버
- 실제 사용자 분포는 중간값 집중 → 초반부터 캐시 히트율 빠르게 상승
- 1~2개월 주기 캐시 초기화로 결과 신선도 유지

### 8.2 동료 케미 보고서 발행 — 실시간 호출

동료 케미 보고서는 두 사람의 DISC 프로필 조합(6⁴ × 6⁴ = 약 168만 가지)으로 캐싱이 비효율적이며, 정확도가 서비스 신뢰도에 직결되므로 **매번 실시간 LLM 호출**을 원칙으로 한다.

사용량은 **코인 시스템**으로 통제한다.

| 항목      | 내용                  |
| --------- | --------------------- |
| 초기 지급 | 가입 시 코인 3개      |
| 충전 주기 | 24시간마다 1개 충전   |
| 최대 보유 | 3개                   |
| 소모량    | 보고서 발행 1회당 1개 |

### 8.3 비용 절감 효과 요약

| 기능                  | 전략                        | 예상 절감 효과                   |
| --------------------- | --------------------------- | -------------------------------- |
| 개인 분석             | Lazy Caching (1,296 버킷)   | LLM 호출 약 99% 절감 (안정화 후) |
| 동료 케미 보고서 발행 | 코인 시스템으로 사용량 통제 | 무제한 호출 대비 대폭 절감       |

## 9. 데이터 모델 개요

### 9.1 주요 엔티티

```
users
  - id (PK)
  - kakao_id (UNIQUE)
  - birth_year
  - gender
  - coins (default: 3)
  - last_coin_charged_at
  - created_at

test_results
  - id (PK)
  - user_id (FK, nullable — 비회원은 NULL)
  - d_score / i_score / s_score / c_score  -- 버킷값 (1~6)
  - birth_year
  - gender
  - created_at

disc_cache
  - d / i / s / c  (복합 PK, 각 1~6)
  - analysis_result (TEXT)
  - created_at

statistics
  - age_group   -- 10대/20대/30대 등
  - gender
  - avg_d / avg_i / avg_s / avg_c
  - sample_count
  - updated_at
```
